# Job Platform User Behavior Analysis  
**채용 플랫폼 사용자 행동 분석 및 리텐션 인사이트**

## 📌 프로젝트 개요  

채용 플랫폼 서비스에서 사용자 행동 로그 데이터를 기반으로  
**신규 유저와 가이드 유저의 행동 패턴**,  
**리텐션(잔존율)**,  
**지원 퍼널 전환율**을 분석하고  
제품/서비스 개선 인사이트를 도출한 프로젝트입니다.

## 📁 데이터 구성  

- `job_platform_analysis.ipynb` – 전체 분석 코드 및 시각화  
- 행동 로그 CSV 파일들 (GitHub 업로드 용량 제한으로 **저장소에는 미포함**, 로컬에서만 사용)  
- `.gitignore`를 통해 대용량 raw 데이터는 버전 관리에서 제외  

## 🔍 주요 분석 내용  

### 1. URL 기반 액션 분류  

- 로그에 기록된 URL을 전처리하여 **사용자 행동 타입(action_type)** 으로 매핑  
- 예시 액션:  
  - 회원가입 페이지 진입 / 회원가입 완료  
  - 공고 상세 조회 (`view_job`)  
  - 회사 정보 조회 (`company_view`)  
  - 지원 단계 진행 (`apply_step1~4`)  
  - 지원서 제출 완료 (`apply_submit`)  

→ 단순 URL 문자열이 아니라, **“사용자가 무엇을 했는지”** 행동 단위로 분석할 수 있도록 구조화.

### 2. 사용자 유형 구분  

- **신규 유저 vs 기존 유저 (`new_user`)**  
- **가이드 유저 vs 비가이드 유저 (`guide_flag`)**  

각 그룹별로  
- 페이지 뷰  
- 공고 조회 수  
- 북마크 수  
- 지원/제출 전환율  
- 30일 리텐션  

등을 비교 분석함.

### 3. 리텐션 분석 (Retention / Cohort)  

- 이벤트 로그를 기반으로 **30일 리텐션** 계산  
- 신규/기존, 가이드/비가이드 그룹별 리텐션 비교  

주요 결과(요약):  

- 전체 유저 30일 리텐션: **약 53.5%**  
- **가이드 유저의 리텐션이 비가이드 대비 확연히 높음**  
- 신규 유저는 첫 방문 후 **초기 이탈이 크게 발생**  

→ **온보딩 구간에서의 경험 설계가 중요**

### 4. 퍼널 분석 (가입 → 조회 → 북마크 → 지원 → 제출)  

가입 이후 전환 퍼널 예시:  

1. 공고 조회 (`view_job`)  
2. 회사 정보 조회 (`company_view`)  
3. 공고 북마크 (`bookmark`)  
4. 지원 단계 진입 (`apply_step`)  
5. 지원서 제출 완료 (`apply_submit`)  

가이드 유저 vs 비가이드 유저 비교 결과:  

- 모든 단계에서 **가이드 유저의 전환율이 더 높음**  
- 특히 북마크/지원/제출 단계에서 차이가 크게 증가  

→ **“가이드 기능이 실제로 전환 퍼널을 밀어주는 역할”** 을 함.

## 🧠 핵심 인사이트  

- **가이드 기능은 강력한 Engagement & 전환 증폭 장치**  
- 신규 유저의 **초기 1~7일 경험이 리텐션에 매우 중요**  
- 북마크(`bookmark`)는 **실제 지원으로 이어지는 강력한 시그널**  
- 행동 로그를 통해 UX 개선에 필요한 정량적 근거 확보  

## 💡 제품/서비스 개선 제안  

1. **신규 유저 온보딩 강화**  
   - 첫 방문 시 관심 직무/지역 기반 공고 추천  
   - “다음에 할 일”을 제안하는 가이드 플로우 강화  

2. **가이드 액션 구조 확장**  
   - 공고/기업 추천, 북마크 유도 문구  
   - “이 공고도 함께 보는 사람들이 많아요” 등 추천 영역 확대  

3. **지원 프로세스 간소화**  
   - 중복 입력 제거, 이력서 자동완성  
   - 모바일 환경에서의 작성 편의 개선  

4. **리텐션 모니터링 체계 구축**  
   - 7일/30일 리텐션 대시보드  
   - A/B 테스트 기반 UI/UX 개선
